How To Update GitHub Repositories

  • First, check what Git branch you are on using command: git status
  • If you are not on your main branch, navigate to it using command: git checkout main
  • Now, from the main branch you want to edit, create a new feature branch using command: git checkout -b feature/feature-name
  • From GitBash, navigate to VS Code using command: code .
  • Add your updates to the appropriate file
  • Next, add your changes to the repository using command: git add -A (NOTE: -A will add all changes made)
  • Next, commit the changes using command: git commit -m "add a description for what was added here"
  • Now make sure the current branch is up to date before opening a pull request using command: git pull origin main
  • Next, push all changes made to your repository using command: git push origin branch name
  • Navigate to your repository on GitHub and click on the "Pull Requests" tab and then click the "New pull request" button
  • Ensure your base branch is set to main and your compare branch is set to whatever branch you just updated and want to add to your main branch
  • Click on the "Create pull request" button
  • Add a title and brief description of the changes made for the pull request
  • If working in a group, at this point you would assign one or two co-workers to review the pull request before merging with the main branch. If working by yourself, you can merge the request without this step
  • Merge your newly updated branch to your main branch by selecting the "Merge Pull Request" button and then confirming the merge request
  • If the merge was successful, delete the compare branch
  • If you had an issue open for the newly merged branch, navigate to the "Issues" tab and close the issue
  • Navigate back to GitBash and update your local main branch using command: git checkout main
  • Then pull the remote main branch from your GitHub repository to your local machine's main branch using command: git pull origin main
